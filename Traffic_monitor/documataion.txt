How each file works
config.py — shared constants so all files stay in sync. Things like UDP_PORT = 5005, CONGESTION_THRESHOLD = 30, SERVER_HOST = "127.0.0.1".
node_sim.py — takes a node_id as a command-line argument. Runs a loop: generates random vehicle_count, cycles through RED/GREEN/YELLOW on a timer, packs everything as JSON, sends over UDP every 1 second. You'll open 3 terminals and run this 3 times with different IDs.
server.py — binds UDP socket on port 5005. Runs a background thread that receives packets, decodes JSON, and updates a shared dictionary (node_data). Also tracks packets-received counter, sequence numbers for loss detection, and timestamps for latency. Exposes this data via a simple module-level object that the dashboard can import and read.
dashboard.py — imports the server module and starts it in a background thread. Builds the Tkinter window with 4 frames arranged in a 2×2 grid using tk.Frame and ttk.Treeview for the table. Uses root.after(1000, refresh) to pull fresh data from the server every second and update all widgets — the table rows, the stat labels, the alert log (a tk.Text widget), and the chart (a tk.Canvas where you draw bars manually with canvas.create_rectangle()).
stress_test.py — spawns N threads each sending UDP packets as fast as possible for 10 seconds. Embeds a sequence number in every packet. Server counts received vs expected. Prints a summary: total sent, total received, loss %, avg latency.
